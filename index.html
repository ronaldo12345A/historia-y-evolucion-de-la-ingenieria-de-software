Construcción de Software

Transformando ideas en código funcional

¿Qué es?

Proceso de crear software significativo y funcional mediante:

  • Codificación
  • Verificación
  • Pruebas unitarias
  • Pruebas de integración
  • Depuración (debugging)

Importancia

  • Área de conocimiento fundamental en el CBOK para Maestrías en Ingeniería de Software
  • Cubre fundamentos, elementos básicos y buenas prácticas
  • Paso esencial que incluye revisión por pares para identificar problemas temprano
  • Contribuye a entregar software de mayor calidad en menos tiempo

Principios Clave

  • Modularidad: Facilita construcción, depuración y reutilización de código
  • Simplicidad en el diseño
  • Pruebas automatizadas
  • Seguir estándares y prácticas establecidas
  • Uso de control de versiones (Git)

Fase de Implementación

Es donde las ideas y diseños se transforman en código. Los programadores escriben código fuente siguiendo estándares establecidos, implementan funcionalidades de forma modular, realizan pruebas unitarias y utilizan sistemas de control de versiones.

Etapas en la Construcción de Software

1

Codificación

Escribir el código fuente según especificaciones de diseño

2

Verificación

Revisar que el código cumple con los requisitos

3

Pruebas

Unitarias y de integración para validar funcionamiento

4

Depuración

Identificar y corregir errores en el código

Rol del Programador

  • Escribe código fuente para instruir a los ordenadores
  • Interpreta directrices de desarrolladores e ingenieros
  • Utiliza lenguajes de programación como C++, Java, etc.
  • Trabaja dentro del ciclo de vida de desarrollo de software
